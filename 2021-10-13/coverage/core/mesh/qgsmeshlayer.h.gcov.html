      43                 :            : /**
      44                 :            :  * \ingroup core
      45                 :            :  *
      46                 :            :  * \brief Represents a mesh layer supporting display of data on structured or unstructured meshes
      47                 :            :  *
      48                 :            :  * The QgsMeshLayer is instantiated by specifying the name of a data provider,
      49                 :            :  * such as mdal, and url defining the specific data set to connect to.
      50                 :            :  * The vector layer constructor in turn instantiates a QgsMeshDataProvider subclass
      51                 :            :  * corresponding to the provider type, and passes it the url. The data provider
      52                 :            :  * connects to the data source.
      53                 :            :  *
      54                 :            :  * The QgsMeshLayer provides a common interface to the different data types. It does not
      55                 :            :  * yet support editing transactions.
      56                 :            :  *
      57                 :            :  * The main data providers supported by QGIS are listed below.
      58                 :            :  *
      59                 :            :  * \section mesh_providers Mesh data providers
      60                 :            :  *
      61                 :            :  * \subsection mesh_memory Memory data providerType (mesh_memory)
      62                 :            :  *
      63                 :            :  * The memory data provider is used to construct in memory data, for example scratch
      64                 :            :  * data. There is no inherent persistent storage of the data. The data source uri is constructed.
      65                 :            :  * Data can be populated by setMesh(const QString &vertices, const QString &faces), where
      66                 :            :  * vertices and faces is comma separated coordinates and connections for mesh.
      67                 :            :  * E.g. to create mesh with one quad and one triangle
      68                 :            :  *
      69                 :            :  * \code{py}
      70                 :            :  *  uri = "1.0, 2.0 \n" \
      71                 :            :  *      "2.0, 2.0 \n" \
      72                 :            :  *      "3.0, 2.0 \n" \
      73                 :            :  *      "2.0, 3.0 \n" \
      74                 :            :  *      "1.0, 3.0 \n" \
      75                 :            :  *      "---" \
      76                 :            :  *      "0, 1, 3, 4 \n" \
      77                 :            :  *      "1, 2, 3 \n"
      78                 :            :  *
      79                 :            :  *  scratchLayer = QgsMeshLayer(uri, "My Scratch layer", "mesh_memory")
      80                 :            :  * \endcode
      81                 :            :  *
      82                 :            :  * \subsection mdal MDAL data provider (mdal)
      83                 :            :  *
      84                 :            :  * Accesses data using the MDAL drivers (https://github.com/lutraconsulting/MDAL). The url
      85                 :            :  * is the MDAL connection string. QGIS must be built with MDAL support to allow this provider.
      86                 :            : 
      87                 :            :  * \code{py}
      88                 :            :  *     uri = "test/land.2dm"
      89                 :            :  *     scratchLayer = QgsMeshLayer(uri, "My Scratch Layer",  "mdal")
      90                 :            :  * \endcode
      91                 :            :  *
      92                 :            :  * \note The API is considered EXPERIMENTAL and can be changed without a notice
      93                 :            :  *
      94                 :            :  * \since QGIS 3.2
      95                 :            :  */
