      89                 :            : /**
      90                 :            :  * \ingroup core
      91                 :            :  * \brief Represents a vector layer which manages a vector based data sets.
      92                 :            :  *
      93                 :            :  * The QgsVectorLayer is instantiated by specifying the name of a data provider,
      94                 :            :  * such as postgres or wfs, and url defining the specific data set to connect to.
      95                 :            :  * The vector layer constructor in turn instantiates a QgsVectorDataProvider subclass
      96                 :            :  * corresponding to the provider type, and passes it the url. The data provider
      97                 :            :  * connects to the data source.
      98                 :            :  *
      99                 :            :  * The QgsVectorLayer provides a common interface to the different data types. It also
     100                 :            :  * manages editing transactions by buffering layer edits until they are written to the
     101                 :            :  * underlying QgsVectorDataProvider. Before edits can be made a call to startEditing()
     102                 :            :  * is required. Any edits made to a QgsVectorLayer are then held in memory only, and
     103                 :            :  * are not written to the underlying QgsVectorDataProvider until a call to commitChanges()
     104                 :            :  * is made. Buffered edits can be rolled back and discarded without altering the
     105                 :            :  * underlying provider by calling rollBack().
     106                 :            :  *
     107                 :            :  *  Sample usage of the QgsVectorLayer class:
     108                 :            :  *
     109                 :            :  * \code{.py}
     110                 :            :  *     uri = "point?crs=epsg:4326&field=id:integer"
     111                 :            :  *     scratchLayer = QgsVectorLayer(uri, "Scratch point layer",  "memory")
     112                 :            :  * \endcode
     113                 :            :  *

     115                 :            :  *
     116                 :            :  * \section providers Vector data providers
     117                 :            :  *
     118                 :            :  * \subsection memory Memory data providerType (memory)
     119                 :            :  *
     120                 :            :  * The memory data provider is used to construct in memory data, for example scratch
     121                 :            :  * data or data generated from spatial operations such as contouring. There is no
     122                 :            :  * inherent persistent storage of the data. The data source uri is constructed. The
     123                 :            :  * url specifies the geometry type ("point", "linestring", "polygon",
     124                 :            :  * "multipoint","multilinestring","multipolygon"), optionally followed by url parameters
     125                 :            :  * as follows:
     126                 :            :  *
     127                 :            :  * - crs=definition
     128                 :            :  *   Defines the coordinate reference system to use for the layer.
     129                 :            :  *   definition is any string accepted by QgsCoordinateReferenceSystem::createFromString()
     130                 :            :  * - index=yes
     131                 :            :  *   Specifies that the layer will be constructed with a spatial index
     132                 :            :  * - field=name:type(length,precision)
     133                 :            :  *   Defines an attribute of the layer. Multiple field parameters can be added
     134                 :            :  *   to the data provider definition. type is one of "integer", "double", "string".
     135                 :            :  *
     136                 :            :  * An example url is "Point?crs=epsg:4326&field=id:integer&field=name:string(20)&index=yes"

     139                 :            :  * loss if there are any non-empty memory layers present. If your memory layer should not
     140                 :            :  * trigger such warning, it is possible to suppress that by setting the following custom variable:
     141                 :            :  *
     142                 :            :  * \code{.py}
     143                 :            :  *     layer.setCustomProperty("skipMemoryLayersCheck", 1)

     232                 :            :  *
     233                 :            :  * Accesses data in a delimited text file, for example CSV files generated by
