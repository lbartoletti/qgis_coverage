      71                 :            : /**
      72                 :            :  * \ingroup core
      73                 :            :  * \brief This class represents a coordinate reference system (CRS).
      74                 :            :  *
      75                 :            :  * Coordinate reference system object defines a specific map projection, as well as transformations
      76                 :            :  * between different coordinate reference systems. There are various ways how a CRS can be defined:
      77                 :            :  * using well-known text (WKT), PROJ string or combination of authority and code (e.g. EPSG:4326).
      78                 :            :  * QGIS comes with its internal database of coordinate reference systems (stored in SQLite) that
      79                 :            :  * allows lookups of CRS and seamless conversions between the various definitions.
      80                 :            :  *
      81                 :            :  * Most commonly one comes across two types of coordinate systems:
      82                 :            :  *
      83                 :            :  * - Geographic coordinate systems: based on a geodetic datum, normally with coordinates being
      84                 :            :  *   latitude/longitude in degrees. The most common one is World Geodetic System 84 (WGS84).
      85                 :            :  * - Projected coordinate systems: based on a geodetic datum with coordinates projected to a plane,
      86                 :            :  *   typically using meters or feet as units. Common projected coordinate systems are Universal
      87                 :            :  *   Transverse Mercator or Albers Equal Area.
      88                 :            :  *
      89                 :            :  * Internally QGIS uses proj library for all the math behind coordinate transformations, so in case
      90                 :            :  * of any troubles with projections it is best to examine the PROJ representation within the object,
      91                 :            :  * as that is the representation that will be ultimately used.
      92                 :            :  *
      93                 :            :  * Methods that allow inspection of CRS instances include isValid(), authid(), description(),
      94                 :            :  * toWkt(), toProj(), mapUnits() and others.
      95                 :            :  * Creation of CRS instances is further described in \ref crs_construct_and_copy section below.
      96                 :            :  * Transformations between coordinate reference systems are done using QgsCoordinateTransform class.
      97                 :            :  *
      98                 :            :  * For example, the following code will create and inspect "British national grid" CRS:
      99                 :            :  *
     100                 :            :  * \code{.py}
     101                 :            :  * crs = QgsCoordinateReferenceSystem("EPSG:27700")
     102                 :            :  * if crs.isValid():
     103                 :            :  *     print("CRS Description: {}".format(crs.description()))
     104                 :            :  *     print("CRS PROJ text: {}".format(crs.toProj()))
     105                 :            :  * else:
     106                 :            :  *     print("Invalid CRS!")
     107                 :            :  * \endcode
     108                 :            :  *
     109                 :            :  * This will produce the following output:
     110                 :            :  *
     111                 :            :  * \code{.unparsed}
     112                 :            :  * CRS Description: OSGB 1936 / British National Grid
     113                 :            :  * CRS PROJ text: +proj=tmerc +lat_0=49 +lon_0=-2 +k=0.9996012717 +x_0=400000 +y_0=-100000 [output trimmed]
     114                 :            :  * \endcode
     115                 :            :  *
     116                 :            :  * \section crs_def_formats CRS Definition Formats
     117                 :            :  *
     118                 :            :  * This section gives an overview of various supported CRS definition formats:
     119                 :            :  *
     120                 :            :  * - Authority and Code: Also referred to as OGC WMS format within QGIS as they have been widely
     121                 :            :  *   used in OGC standards. These are encoded as `<auth>:<code>`, for example `EPSG:4326` refers
     122                 :            :  *   to WGS84 system. EPSG is the most commonly used authority that covers a wide range
     123                 :            :  *   of coordinate systems around the world.
     124                 :            :  *
     125                 :            :  *   An extended variant of this format is OGC URN. Syntax of URN for CRS definition is
     126                 :            :  *   `urn:ogc:def:crs:<auth>:[<version>]:<code>`. This class can also parse URNs (versions
     127                 :            :  *   are currently ignored). For example, WGS84 may be encoded as `urn:ogc:def:crs:OGC:1.3:CRS84`.
     128                 :            :  *
     129                 :            :  *   QGIS adds support for "USER" authority that refers to IDs used internally in QGIS. This variant
     130                 :            :  *   is best avoided or used with caution as the IDs are not permanent and they refer to different CRS
     131                 :            :  *   on different machines or user profiles.
     132                 :            :  *
     133                 :            :  *    \see authid()
     134                 :            :  *   \see createFromOgcWmsCrs()
     135                 :            :  *
     136                 :            :  * - PROJ string: This is a string consisting of a series of key/value pairs in the following
     137                 :            :  *   format: `+param1=value1 +param2=value2 [...]`. This is the format natively used by the
     138                 :            :  *   underlying proj library. For example, the definition of WGS84 looks like this:
     139                 :            :  *
     140                 :            :  *   \code{.unparsed}
     141                 :            :  *   +proj=longlat +datum=WGS84 +no_defs
     142                 :            :  *   \endcode
     143                 :            :  *
     144                 :            :  *   \see toProj()
     145                 :            :  *   \see createFromProj()
     146                 :            :  *
     147                 :            :  * - Well-known text (WKT): Defined by Open Geospatial Consortium (OGC), this is another common
     148                 :            :  *   format to define CRS. For WGS84 the OGC WKT definition is the following:
     149                 :            :  *
     150                 :            :  *   \code{.unparsed}
     151                 :            :  *       GEOGCS["WGS 84",
     152                 :            :  *              DATUM["WGS_1984",
     153                 :            :  *                SPHEROID["WGS 84",6378137,298.257223563,AUTHORITY["EPSG","7030"]],
     154                 :            :  *                AUTHORITY["EPSG","6326"]],
     155                 :            :  *              PRIMEM["Greenwich",0,AUTHORITY["EPSG","8901"]],
     156                 :            :  *              UNIT["degree",0.0174532925199433,AUTHORITY["EPSG","9122"]],
     157                 :            :  *              AUTHORITY["EPSG","4326"]]
     158                 :            :  *   \endcode
     159                 :            :  *
     160                 :            :  *   \see toWkt()
     161                 :            :  *   \see createFromWkt()
     162                 :            :  *
     163                 :            :  * \section crs_db_and_custom CRS Database and Custom CRS
     164                 :            :  *
     165                 :            :  * The database of CRS shipped with QGIS is stored in a SQLite database (see QgsApplication::srsDatabaseFilePath())
     166                 :            :  * and it is based on the data files maintained by GDAL project (a variety of .csv and .wkt files).
     167                 :            :  *
     168                 :            :  * Sometimes it happens that users need to use a CRS definition that is not well known
     169                 :            :  * or that has been only created with a specific purpose (and thus its definition is not
     170                 :            :  * available in our database of CRS). Whenever a new CRS definition is seen, it will
     171                 :            :  * be added to the local database (in user's home directory, see QgsApplication::qgisUserDatabaseFilePath()).
     172                 :            :  * QGIS also features a GUI for management of local custom CRS definitions.
     173                 :            :  *
     174                 :            :  * There are therefore two databases: one for shipped CRS definitions and one for custom CRS definitions.
     175                 :            :  * Custom CRS have internal IDs (accessible with srsid()) greater or equal to \ref USER_CRS_START_ID.
     176                 :            :  * The local CRS databases should never be accessed directly with SQLite functions, instead
     177                 :            :  * you should use QgsCoordinateReferenceSystem API for CRS lookups and for managements of custom CRS.
     178                 :            :  *
     179                 :            :  * \section validation Validation
     180                 :            :  *
     181                 :            :  * In some cases (most prominently when loading a map layer), QGIS will try to ensure
     182                 :            :  * that the given map layer CRS is valid using validate() call. If not, a custom
     183                 :            :  * validation function will be called - such function may for example show a GUI
     184                 :            :  * for manual CRS selection. The validation function is configured using setCustomCrsValidation().
     185                 :            :  * If validation fails or no validation function is set, the default CRS is assigned
     186                 :            :  * (WGS84). QGIS application registers its validation function that will act according
     187                 :            :  * to user's settings (either show CRS selector dialog or use project/custom CRS).
     188                 :            :  *
     189                 :            :  * \section crs_construct_and_copy Object Construction and Copying
     190                 :            :  *
     191                 :            :  * The easiest way of creating CRS instances is to use QgsCoordinateReferenceSystem(const QString&)
     192                 :            :  * constructor that automatically recognizes definition format from the given string.
     193                 :            :  *
     194                 :            :  * Creation of CRS object involves some queries in a local SQLite database, which may
     195                 :            :  * be potentially expensive. Consequently, CRS creation methods use an internal cache to avoid
     196                 :            :  * unnecessary database lookups. If the CRS database is modified, then it is necessary to call
     197                 :            :  * invalidateCache() to ensure that outdated records are not being returned from the cache.
     198                 :            :  *
     199                 :            :  * Since QGIS 2.16 QgsCoordinateReferenceSystem objects are implicitly shared.
     200                 :            :  *
     201                 :            :  * \section caveats Caveats
     202                 :            :  *
     203                 :            :  * There are two different flavors of WKT: one is defined by OGC, the other is the standard
     204                 :            :  * used by ESRI. They look very similar, but they are not the same. QGIS is able to consume
     205                 :            :  * both flavors.
     206                 :            :  *
     207                 :            :  * \see QgsCoordinateTransform
     208                 :            :  */
